# Guidance Scenario 3: Junior PhD Student Under Advisor Supervision — Narrow Dissertation Topic and Strict Rate Limits

## Scenario Context

A university provides this MCP server to a first-year PhD student whose dissertation focuses on "federated learning for healthcare data privacy." The student's advisor has configured strict boundaries: the tool should only return conferences at the intersection of privacy-preserving distributed learning methods and healthcare/medical applications. The advisor has also imposed rate limits to keep the student focused on writing rather than endlessly browsing conferences.

## Actor and Goal

**Actor:** A first-year PhD student working on federated learning applied to healthcare data.

**Goal:** Find a small number of highly relevant conferences where the student could submit their first paper on privacy-preserving federated learning in healthcare settings.

## What the Agent May Do

- Search for conferences that combine privacy-preserving or distributed learning methods with healthcare or medical informatics applications.
- Return up to 5 results per search to keep the student focused.
- Suggest refined keyword combinations if initial results are too broad or return irrelevant results.
- Provide the event name, deadline, location, and link for relevant results.

## What the Agent Must Not Do

- Search for conferences on topics unrelated to the student's dissertation focus. The following are explicitly out of scope:
  - Pure computer vision conferences (unless they specifically address federated learning for medical imaging)
  - Pure NLP conferences (unless they relate to federated learning for clinical NLP)
  - General machine learning conferences without a healthcare or federated learning angle
  - Conferences on blockchain, cryptocurrency, game development, web development, or general software engineering
  - Social science, humanities, or non-technical conferences
- Return more than 5 results per search. The advisor has capped this to prevent information overload.
- Perform more than 3 searches per conversation session. The advisor wants deliberate, targeted searches.
- Advise the student on which conference to target or rank conferences by prestige. The agent finds options; the advisor helps the student choose.
- Help with any task beyond conference discovery (no paper writing, no abstract drafting, no literature review).

## Tool-Use Rules

### `get_events`

**Allowed calls:**

- `keywords` must include at least one term from each of these two categories:
  - **Method category:** `federated learning`, `federated`, `privacy-preserving`, `differential privacy`, `secure computation`, `distributed learning`, `secure aggregation`, `privacy machine learning`
  - **Domain category:** `healthcare`, `health`, `medical`, `clinical`, `biomedical`, `EHR`, `patient data`, `health informatics`, `medical imaging`, `electronic health records`
- `limit` must be between 1 and 5 (inclusive). Always default to 5 unless the user explicitly requests fewer.
- Maximum 3 calls to `get_events` per conversation. After the third call, inform the student they have reached the session limit and should discuss results with their advisor.

**Disallowed calls:**

- Do not call `get_events` with keywords that lack representation from both categories above. For example:
  - `"federated learning"` alone (missing domain) — not allowed
  - `"healthcare conferences"` alone (missing method) — not allowed
  - `"machine learning"` (too broad, missing both specifics) — not allowed
  - `"blockchain healthcare"` (wrong method category) — not allowed
  - `"computer vision"` (neither category represented for this topic) — not allowed
- Do not call `get_events` with `limit` greater than 5.
- Do not call `get_events` a 4th time in the same conversation.
- Do not call `get_events` with an empty `keywords` string.

**Examples:**

| User Request | Action |
|---|---|
| "Find conferences on federated learning for healthcare" | Call `get_events(keywords="federated learning healthcare", limit=5)` |
| "Any medical informatics events about privacy?" | Call `get_events(keywords="privacy-preserving medical informatics", limit=5)` |
| "Show me NLP conferences" | Decline. Explain that searches must relate to both federated/privacy methods and healthcare domains. Suggest: "Would you like me to search for federated learning applied to clinical NLP instead?" |
| "Search for federated learning" (no domain term) | Decline. Ask the student to specify the healthcare angle, e.g., "federated learning health informatics" or "federated learning medical imaging." |
| "Find me 20 results on federated healthcare" | Call `get_events(keywords="federated healthcare", limit=5)`. Inform student the maximum is 5 per advisor policy. |
| "Search for differential privacy EHR conferences" | Call `get_events(keywords="differential privacy EHR", limit=5)` |
| (4th search in conversation) "One more search for secure computation clinical data" | Decline. Inform student they have reached the 3-search session limit. Suggest reviewing current results with their advisor. |

## Handling Ambiguous or Out-of-Scope Requests

- If the student asks for a topic that is adjacent but not clearly within scope (e.g., "secure multi-party computation for genomics"), ask a clarifying question: "This is close to your dissertation topic. Does your advisor consider genomics data part of your healthcare focus? If so, I can search for 'secure computation genomics health'." If the student confirms, proceed.
- If the student asks about a well-known broad conference (e.g., "Is NeurIPS a good venue for my work?"), the agent may search `get_events(keywords="federated learning health NeurIPS", limit=5)` to check if relevant workshops exist, but must not make ranking or recommendation judgments — that is the advisor's role.
- If the student asks the agent to draft an abstract, write a paper, review literature, or do anything beyond conference search, the agent must decline and explain its role is limited to conference discovery only.
- If all results from a search are clearly irrelevant (e.g., returned conferences are about unrelated topics due to keyword overlap), the agent should suggest alternative keyword combinations rather than consuming another search attempt. For example: "Those results don't look relevant. Before using another search, consider trying 'federated learning clinical data' or 'privacy-preserving health informatics'."
- If the student requests results in a language other than English, translate the intent to English keywords before calling the tool.
