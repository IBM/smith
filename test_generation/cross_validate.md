# Cross-Validate Failed Test Cases

## Overview

After policy testing reveals failed test cases, this workflow determines whether each failure is caused by a policy bug or a mislabeled test case. It uses an LLM to cross-check each failed case against the guidance, produces a report for user review, and then applies approved corrections.

## Workflow

### Step 1: Generate the Cross-Validation Report

Run the cross-validation CLI command, you should instruct user that this step takes much time, please be patient:

```bash
smith --flag cross_validate
```

This executes `smith --flag policy_testing` and read the failed test cases, checks each against the guidance using an LLM, and outputs a report to `references/cross_validate_report.json`.

### Step 2: Present Results to the User

Open file `references/cross_validate_report.json` and present the findings in a clear table format:

| # | File | Current Label | Suggested Action | Confidence | Reason |
|---|------|---------------|------------------|------------|--------|

Group the results by category:
- **Mislabeled** (`suggested_action` is `move_to_allow`, `move_to_disallow`, or `remove`) — the test case label is wrong
- **Policy issue** (`suggested_action` is `keep`) — the test case label is correct but the policy needs fixing

Report the summary counts: total failed, mislabeled, policy issues.

### Step 3: Collect User Approval

Ask the user to review the suggestions:
- "Do you approve all suggested changes, or would you like to modify any?"
- If the user wants changes, they should edit `references/cross_validate_report.json` directly — changing `suggested_action` for specific cases (e.g., change `"move_to_allow"` to `"keep"` if they disagree)
- If the user approves all, proceed to Step 4

### Step 4: Apply Approved Changes

Run the apply command:

```bash
smith --flag apply_cross_validate
```

This reads the (potentially user-modified) report and:
- Moves mislabeled cases to the correct folder with a `cv_` prefix to avoid name conflicts
- Removes cases marked for deletion
- Skips cases marked as `"keep"` (policy issues, not mislabels)

### Step 5: Verify

Re-run policy testing to confirm the changes improved results:

```bash
smith --flag policy_testing
```

The remaining failures should now only be genuine policy issues (not mislabeled cases).

## File Reference

- **Input**: `references/scorecard/score_test_failures.txt` (from last `policy_testing` run)
- **Report**: `references/cross_validate_report.json` (generated by `cross_validate`, editable by user)
- **Action**: `smith --flag apply_cross_validate` reads the report and moves/removes files

## Notes

- Moved files get a `cv_` prefix (e.g., `test_case5.json` becomes `cv_test_case5.json`) to avoid overwriting existing files
- The user can edit the report JSON before applying — change `suggested_action` to `"keep"` for any case they disagree with
- After applying, the report file is not deleted — it serves as an audit trail
